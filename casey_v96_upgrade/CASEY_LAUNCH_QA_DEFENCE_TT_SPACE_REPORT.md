## Issue found and patched

The second QA pass found one important issue:

- Earth-based “secure satellite control centre” defence projects were being interpreted as Space because of the word “satellite”.

This has been patched.

CASEY now correctly classifies Earth-based secure satellite/control/mission operations facilities as:

- Earth
- Defence / Secure Mission Infrastructure

## Output realism assessment

The outputs are strong enough for a public demo launch as **first-pass project-controls intelligence**.

They are suitable for:

- early class estimate demonstration
- concept-stage intelligence
- showing cost/schedule/risk logic
- showing CASEY’s value proposition
- lead capture
- “wow” demo experience

They should not be presented as:

- certified QS estimate
- validated benchmarked capital cost
- formal tender estimate
- engineer-stamped programme
- professional advice without review

Best public wording:

> CASEY generates a first-pass project intelligence pack for early-stage cost, schedule and risk thinking.

## Input governance added

The demo now checks for:

- valid email
- minimum brief quality
- project asset type
- location / operating environment
- size or capacity
- project stage
- main concern
- Earth/Space consistency
- conflicting orbital/location language

If the input is too vague, CASEY does **not** waste the user’s free run. It asks for a stronger brief.

## Learning system

CASEY still does **not** self-train live. This is intentional.

It logs:

- user brief
- generated output
- one-shot run ID
- feedback rating
- feedback comment

Correct improvement loop:

1. user runs demo
2. CASEY stores prompt/output
3. user leaves feedback
4. weak runs are reviewed weekly
5. rules/prompts/examples are improved
6. better version is redeployed

This is safer than uncontrolled live self-training.

## Does this need an OpenAI key?

The current tested engine can run from CASEY’s internal deterministic project-controls engine.

To make the public demo feel even more premium later, add `OPENAI_API_KEY` on Render and use it for:

- narrative polish

- board-report rewriting

Do not expose any AI provider name to users. The product should say:

- Generated by CASEY

not:

- Generated by ChatGPT / Claude

## ChatGPT vs Claude recommendation

For your use case:

- OpenAI / GPT-4.1 or GPT-4o is best for structured API output and backend integration.
- Claude is excellent for long-form advisory narrative.
- Best production architecture: CASEY deterministic engine → OpenAI structured report generation → CASEY quality checker → branded final report.
