# CASEY V118 Final Demo Release Notes

This build applies the final export realism refinements requested for the Earth + Space demo.

## Export upgrades

### XER / schedule sophistication
- Relationship types added: FS, SS and FF-style Primavera relationship coding.
- Lags and leads added to schedule relationships.
- Schedule CSV now includes WBS path, relationship type, lag/lead days, calendar, primary resource, resource loading FTE, near-critical flag and float days.
- XER now injects richer predecessor logic, lag hours, near-critical float and scenario-specific task metadata.
- Calendars are assigned by activity phase, including business, delivery, commissioning, launch-window and surface-operations style calendars.
- Resource loading is generated by phase so the exported schedule feels more planner-grade.

### Tornado / QCRA / QSRA realism
- Risk curves use asymmetric tail behaviour rather than perfectly clean symmetric values.
- Risk register now exposes upside/downside deltas separately.
- Tornado sheets prioritise downside exposure and show more irregular driver behaviour.
- P-curves include controlled irregularity to avoid a hand-authored or overly clean simulation feel.

### Risk ownership depth
- Risk register now includes named owners, response status, mitigation confidence, residual exposure, target review date and escalation threshold.
- A new Response Plan sheet turns the register into an operational governance artifact rather than a static list.
- Model audit JSON includes the enhanced risk and schedule fields for traceability.

## Demo positioning
This remains a demo-watermarked first-pass intelligence system, not a certified commercial estimate, formal PRA model or contractual schedule.
